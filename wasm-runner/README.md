# Spartan WASM Runner

Executes WASM files generated by the Spartan VM LLVM codegen and outputs witnesses and constraints.

## Build

```bash
npm install
npm run build
```

## Usage

```bash
node dist/index.js <wasm-file> [options]
```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `-i, --input <path>` | Path to Prover.toml input file | `./Prover.toml` |
| `-o, --output <path>` | Output JSON file path | `./output.json` |

### Example

```bash
node dist/index.js ./witgen.wasm -i ./Prover.toml -o ./output.json
```

## Input Format

The runner expects a `Prover.toml` file with field element inputs:

```toml
x = "2"
y = "123"
```

It also requires a metadata file `<wasm-file>.meta.json` alongside the WASM file:

```json
{
  "witnessCount": 1000003,
  "constraintCount": 1000001,
  "parameterCount": 2,
  "parameterNames": ["x", "y"]
}
```

## Output Format

```json
{
  "witnesses": ["0x0000...0002", "0x0000...0004", ...],
  "constraints": {
    "a": ["0x...", ...],
    "b": ["0x...", ...],
    "c": ["0x...", ...]
  }
}
```

Field elements are output as 64-character hex strings (256-bit values).
